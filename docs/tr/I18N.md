# Diller (TR / EN)

Bu doküman, uygulamanın her şeyi hem Türkçe hem İngilizce nasıl gösterdiğini
anlatır — arayüzü, tarihleri, sıralamayı ve bu dokümantasyon sayfalarını.
Herkes için manşet: **tek tık tüm uygulamayı değiştirir**; alttaki bölümler
geliştiricilere bu makinenin her parçasını gösterir.

---

## Tek anahtar, her şey peşinden gelir

TR/EN çipi `i18n.changeLanguage`'i çağırır ve tek bir dinleyici değişikliği
ilgilenen her sisteme dağıtır (`src/plugins/i18n.ts`, gönderildiği gibi):

```ts
function syncLanguageCompanions(language: string): void {
  const base = language.split('-')[0]
  setPrimeReactLocale(base)   // bileşen metinleri (paginator, takvim, …)
  setDayjsLocale(base)        // tarih biçimleme
  if (typeof document !== 'undefined') {
    document.documentElement.lang = base   // erişilebilirlik: <html lang>
  }
}

i18n.on('languageChanged', syncLanguageCompanions)
```

```
AppLanguageSwitcher çipi
  → i18n.changeLanguage('en' | 'tr')
      → react-i18next her t() metnini yeniden çizer
      → PrimeReact locale'i döner (kendi bileşen metinleri)
      → Day.js locale'i döner (tarih çıktısı)
      → <html lang> güncellenir
      → doküman görüntüleyici içeriği yeni dilde yeniden sorgular
```

Varsayılan dil Türkçedir; İngilizce geri-düşüştür.

## Kelimeler nerede yaşar

Görünen her cümle `src/locales/tr.json` ve `en.json`'da yaşar — her zaman aynı
anahtar kümesiyle (iki dosya ayrışırsa bir `node:test` kırmızıya düşer). Bir
cümleyi doğrudan koda yazmak lint tarafından reddedilir; lint'in göremediği
boşluklar (toast'lar, doğrulama mesajları) **tiplerle** kapatılır: o API'ler
yalnız `TranslationKey` kabul eder — `en.json`'dan türetilen her anahtar
yolunun union'ı. Anahtardaki bir yazım hatası derleme hatasıdır.

```ts
t('docs.title')          // ✓ locale şekline karşı denetlenir
t('docs.tittle')         // ✗ derleme hatası
notify.success('Saved!') // ✗ derleme hatası — TranslationKey değil
```

## Veride kodlar, locale'lerde etiketler

Enum-benzeri değerler kararlı İngilizce **kod** olarak saklanır;
**etiketleri** locale girdileridir. Kodlar locale dosyalarında yaşar ve hasta
modeli (`PatientStatus`, `PatientPriority`, `PatientDepartment` union'ları)
tam olarak aynı kodları taşır — mapper, API'nin Türkçe görünen değerlerini
seed anında onlara çevirir:

| Kod (saklanan) | TR etiket | EN etiket |
| --- | --- | --- |
| `waiting` | Bekliyor | Waiting |
| `inExamination` | Muayenede | In examination |
| `completed` | Tamamlandı | Completed |
| `cancelled` | İptal edildi | Cancelled |
| `urgent` / `normal` | Acil / Normal | Urgent / Normal |
| `internalMedicine`, `cardiology`, … | Dahiliye, Kardiyoloji, … | Internal medicine, Cardiology, … |

Yani bir durum `t('patients.status.waiting')` olarak çizilir — asla gömülü bir
kelime olarak değil. Kan grupları ham gösterimi anahtar olarak tutar (`0+`,
`A-`, …); EN etiketler O harfini kullanır, TR sıfırı korur.

## İki dilli veri alanları

Bazı hasta alanları **verinin kendisinde** iki dilde bulunur (API'de
`note_tr` / `note_en`, `diagnosis_tr` / `diagnosis_en`). Gösterim, Türkçe
geri-düşüşle etkin dili seçer — küçücük bir saf fonksiyon
(`src/lib/pickLocalized.ts`, gönderildiği gibi):

```ts
export const pickLocalized = (tr: string, en: string, language: string): string => {
  const base = language.split('-')[0]
  return base === 'en' ? en || tr : tr || en
}
```

İşlenmiş örnek: `noteTr = "Kontrol randevusu"`, `noteEn = ""` → İngilizce
arayüz "Kontrol randevusu" gösterir (geri-düşüş), Türkçe arayüz aynısını.
Formlar iki dil alanını yan yana gösterecek, sekme yok (1.3).

## Türkçe bilen metin

Türkçenin, saf string karşılaştırmasının yanlış yaptığı harfleri ve büyük/küçük
kuralları vardır (`İ`/`i`, `I`/`ı`, `ğ`, `ş`). Tüm arama ve sıralama
`src/lib/text.ts` üzerinden geçer (gönderildiği gibi):

```ts
export function normalizeTurkish(value: string): string {
  return value.normalize('NFC').toLocaleLowerCase('tr')
}

const turkishCollator = new Intl.Collator('tr', { numeric: true })

export function compareTurkish(a: string, b: string): number {
  return turkishCollator.compare(a, b)
}
```

`plugins/primereact.ts`'te kayıtlı `nfcContains` filtresi, `turkishIncludes`'u
her `AppDataTable` arama kutusuna bağlar; "ıs" araması "Işık"ı bulur.
Collator'ın `numeric: true`'su ayrıca "2"yi "10"dan önce sıralar.

## Tarihler

Tarihler veride ISO string'dir ve **yalnız** `formatDate` üzerinden biçimlenir
(`src/lib/date.ts`, gönderildiği gibi) — dağınık `toLocaleString` çağrıları
yasaktır:

```ts
export function formatDate(value: string | null | undefined, pattern = 'L'): string {
  if (!value) {
    return ''
  }
  const parsed = dayjs(value)
  return parsed.isValid() ? parsed.format(pattern) : ''
}
```

| Alan | Desen | TR çıktı | EN çıktı |
| --- | --- | --- | --- |
| doğum tarihi | `L` | 24.03.1987 | 03/24/1987 |
| randevu | `LLL` | 5 Haziran 2026 14:30 | June 5, 2026 2:30 PM |

Geçersiz ya da eksik girdi boş string olarak çizilir, asla "Invalid Date"
değil.

## Bu dokümanlar da iki dilli

Her dokümantasyon sayfası `docs/en/<AD>.md` ve `docs/tr/<AD>.md` olarak
bulunur; görüntüleyici etkin dilin dosyasını seçer ve dil değiştirdiğinizde
canlı takaslar. Tek istisna Değişiklik Günlüğü'dür — sürüm aracı onu
İngilizce üretir ve iki dilde de İngilizce gösterilir. Düzyazı çevrilir; kod,
dosya adları ve tanımlayıcılar iki dilde de İngilizce kalır.
