# Languages (TR / EN)

This document explains how the application shows everything in both Turkish
and English: the interface, the dates, the sorting, and even these
documentation pages. It is for anyone curious how the two-language experience
works; the enforcement details matter mostly to developers.

> This is an early skeleton. Each section will be expanded with full detail
> in an upcoming documentation pass.

## One switch, everything follows

The TR/EN chip in the top bar switches the language of the whole application
in one click: every label, every component, every date format, and the
documentation you are reading now.

## Where the words live

Every visible sentence lives in two translation files — one Turkish, one
English — that always contain the same set of entries. Writing a visible
sentence directly into the code is automatically rejected.

## Bilingual data

Some patient fields (notes, diagnoses) exist in both languages in the data
itself. The screen shows the one matching your language, falling back to
Turkish when a translation is missing.

## Turkish-aware text

Searching and sorting understand Turkish letters: "ı" and "i" are treated
correctly, and the alphabet sorts the Turkish way.

## Dates

Dates are stored in one neutral format and displayed in the format natural to
the active language.
